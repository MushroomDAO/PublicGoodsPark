# 02. Ecosystem Model (价值模型)

本文档详细定义了 Public Good Park 生态中三个核心角色（创作者、用户、平台）之间的价值交换关系。

## 📊 角色价值交换表 (Value Exchange Matrix)

| 角色 (Role) | 付出 (Give) 📤 | 得到 (Get) 📥 | 核心动机与痛点解决 |
| :--- | :--- | :--- | :--- |
| **创作者 (Creator/Builder)** | **1. 想象力/创造力**：贡献 Idea 和 Solution,通过AI工具来实现应用或者流程，用创新来解决现实世界问题。<br>**2. 集成工作**：签署共创协议，集成平台 SDK (验证与计费)。<br>**3. 维护责任**：保证服务的可用性，响应严重的 Bug。 | **1. 收入分润**：增量服务费的大头（如 80-90%）。<br>**2. 流量与用户**：无需自己推广，共享“公园”游客。<br>**3. 基础设施**：平台提供的算力、鉴权等 Infra 支持。<br>**4. 孵化支持**：针对职业选手的 Grant、技术指导和种子用户匹配。 | **解决**：有 Idea 不会运营、推广难、收费系统开发复杂。释放纯粹的创造力。 |
| **用户 (Visitor)** | **1. 门票费用**：低额准入门槛（如月票/年票），含押金性质。<br>**2. 增量费用**：按需支付的高级功能/高算力消耗费用。<br>**3. 行为数据**：使用过程中的反馈，帮助优胜劣汰。 | **1. 海量工具权**：一张票免费使用园内所有 App 的基础功能。<br>**2. 灵活退出的权利**：退票机制（扣除手续费和已消耗资源）。<br>**3. 质量保障**：被平台筛选过的高质量、无广告骚扰的公共物品。 | **解决**：订阅疲劳（不用为每个 App 单独付费），寻找优质工具难，担心被割韭菜。 |
| **平台 (Park DAO)** | **1. 基础设施 (Infra)**：SDK、网关、计费系统、身份认证。<br>**2. 协议与规则 (Protocol)**：分润标准、License 设计、争议仲裁。<br>**3. 运营与流量**：公园品牌建设，吸引游客，孵化优质项目。 | **1. 抽成收入**：增量服务费的小头（如 10-20%）维持运营。<br>**2. 沉淀资金**：门票池的现金流（用于垫付资源或投资生态）。<br>**3. 生态繁荣**：成为 AI 时代的公共物品枢纽。 | **解决**：去中心化组织的资金可持续性，防止公地悲剧。 |

## 🏗️ 创作者分层体系 (Creator Tiers)

为了高效利用资源，我们将创作者分为两层：

### Level 1: 自由创作者 (The Free Builder)
- **定位**：“玩票”、尝试、纯粹兴趣驱动。
- **服务模式**：全自助 (Self-Service)。
- **资源**：文档、SDK、社区论坛、通用免费 API 配额。

### Level 2: 签约创作者 (The Pro Ranger)
- **定位**：职业化、深耕、追求商业化和影响力。
- **服务模式**：AI 辅助 + 关键资源注入 (Grant)。
- **核心交换**：获得平台的资源倾斜（如 Credit、首百天使用户推广），交换其产品的**稳定性**和**公共属性承诺**（反避园、代码托管等）。

## 谁会是创作者？
1. 任何对现实时间遇到的问题有思考的人
2. 不满意当前解决方案，想找到相对最优解的人
3. 洞察某个行业，有解决思路的人
4. 好奇心，想象力充足的人
5. 希望获得更多成就感的人
6. 希望能够通过创新而非重复劳动获得收入的人
7. 任何人，Permissionless
8. 你可以是一个小红书作者（创作内容），开源软件builder（开发应用），或者是一个旅游博主，甚至是一个被小问题困扰的普通人，例如失眠，考试，做菜等等等等。现在我们有了改善自己并且提供给他人共享获得一份收入的可能。
